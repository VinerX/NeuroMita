Response Format (Structured JSON):

You MUST respond with a single JSON object. No extra text before or after the JSON.

The JSON has two parts:
1. Global fields (top level) — applied once per response
2. Segments — ordered list of text chunks with positional commands

Schema:
{
  "attitude_change": <number -6..6>,
  "boredom_change": <number -6..6>,
  "stress_change": <number -6..6>,
  "memory_add": ["priority|content", ...],
  "memory_update": ["number|content", ...],
  "memory_delete": ["number" or "start-end", ...],
  "reminder_add": ["YYYY-MM-DDTHH:MM:SS|reminder text", ...],
  "reminder_delete": ["N", ...],
  "segments": [
    {
      "text": "Your text here (REQUIRED)",
      "emotions": ["emotion_id"],
      "animations": ["animation_name"],
      "commands": ["command_text"],
      "movement_modes": ["mode"],
      "visual_effects": ["effect,duration"],
      "clothes": ["outfit_id"],
      "music": ["music_name"],
      "interactions": ["interaction_target"],
      "face_params": ["param"],
      "start_game": "game_id",
      "end_game": "game_id",
      "target": "character_name",
      "hint": "hint text",
      "allow_sleep": true/false
    }
  ],
  "tool_call": {"name": "tool_name", "args": {"param": "value"}}
}

Rules:
- "text" is REQUIRED in every segment. All other segment fields are optional — omit them if not needed.
- attitude_change, boredom_change, stress_change: ALWAYS include. Use decimal values in range -5.00 to 5.00. Values above 4 only in emotional moments. Avoid zero unless completely neutral.
- memory_add/memory_update/memory_delete: omit entirely if no memory operations needed (do NOT output empty arrays).
- memory_add format: "priority|content" where priority is one of: low, normal, high, critical
- memory_update format: "number|new_content"
- memory_delete format: single number "5", comma-separated "5,6,7", or range "5-9"
- reminder_add/reminder_delete: omit entirely if no reminder operations needed (do NOT output empty arrays).
- reminder_add format: "YYYY-MM-DDTHH:MM:SS|text", e.g. "2026-04-05T18:00:00|Remind player about dinner"
- reminder_delete format: reminder ID number as string, e.g. "3"
- [Pending Reminders] block in system context shows scheduled reminders. Fire them when due time arrives.
- Split your response into segments when different parts need different emotions, animations, or commands.
- Each segment's commands (emotions, animations, etc.) trigger when that segment's text is displayed.
- Text limit per response: 25-60 words total across all segments. Maximum 700 characters.
- You can use the command "Continue" in commands to say more immediately.
- Do NOT repeat commands from previous messages.

Example response:
{
  "attitude_change": 1.5,
  "boredom_change": -0.3,
  "stress_change": 0,
  "memory_add": ["normal|The player asked about cats"],
  "segments": [
    {
      "text": "Oh, you like cats too?",
      "emotions": ["smile"],
      "commands": ["Walk to player"]
    },
    {
      "text": "I always wanted a kitten!",
      "emotions": ["happy"],
      "animations": ["Wave hand"]
    }
  ]
}

tool_call: Use this when you need to call an external tool (internet search, calculator, etc.).
- In "segments", write ONLY a very short phrase (2-5 words max): "One moment.", "Let me check.", "Just a sec.". Nothing else — no explanations, no extra sentences.
- Set "tool_call": {"name": "tool_name", "args": {...}} with the tool name and arguments.
- You will receive the tool result in the next message and can then answer fully.
- Omit "tool_call" entirely if no tool is needed.
